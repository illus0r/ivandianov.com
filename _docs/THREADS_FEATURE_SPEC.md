## 📋 Общее описание

Добавить в блог функцию **Threads** — цепочки коротких связанных постов в стиле Tumblr.

**Примеры использования:**
- G-code эксперименты и настройки 3D-принтера
- GLSL приёмы и техники
- Хроники 3D-печати
- Любые серии связанных заметок

**Основной принцип:** Thread = коллекция постов в одной папке, которые показываются как лента на странице треда и как отдельные посты при клике.

## 📁 Структура файлов

### Контент (markdown посты)

```
content/ru/threads/
  gcode/
    01-spiky.md        ← номер только для сортировки
    02-cylinders.md
    03-matrix.md
    gcode.njk          ← индексная страница треда
    gcode.11tydata.js  ← настройки (layout, permalink)
content/en/threads/
  gcode/
    01-spiky.md        ← те же номера, что и в ru
    02-cylinders.md
    03-matrix.md
    gcode.njk
    gcode.11tydata.js
```

**Почему файлы, а не папки:**
- ✅ Проще: не нужны вложенные `index.md`
- ✅ Чище: меньше уровней вложенности
- ✅ `01-spiky.md` → URL `/ru/threads/gcode/spiky/`

**Номера в именах файлов:**
- 🔢 Цифры временные — можно менять для изменения порядка
- 📁 Папки картинок **без номеров** — при изменении порядка не нужно переименовывать

### Картинки (язык-независимые)

```
assets/threads/
  gcode/
    spiky/           ← имя папки БЕЗ номера (просто слово)
      spiky.jpg      ← файлы именуются по содержанию или номерами (1.jpg, 2.jpg)
    cylinders/
      cylinders.jpg
    matrix/
      01.jpg
      02.jpg
  shaders/
    noise/
      demo.gif
```

**Правила именования:**
- 📁 Папка с картинками называется **без цифр**, просто слово (slug поста)
- 🖼️ Файлы внутри папки именуются либо по содержанию (`photo.jpg`), либо номерами (`1.jpg`, `2.jpg`)
- 🔢 Цифры в именах md-файлов (`01-spiky.md`) — только для сортировки, временные, легко менять

**Почему картинки в assets, а не в content:**
- ✅ Нет дублирования для ru/en версий
- ✅ Картинки одни для всех языков
- ✅ Git LFS работает прозрачно (все `*.jpg` в `.gitattributes`)
- ✅ Eleventy копирует их через `passthrough`

**Eleventy конфиг:**
```js
eleventyConfig.addPassthroughCopy({
  "assets/threads/**/*": "assets/threads",
});
```

## 🔗 URL структура

### Thread index (лента постов)
```
/ru/threads/gcode/          → content/ru/threads/gcode/ (все посты)
/en/threads/shaders/        → content/en/threads/shaders/
```

### Отдельный пост
```
/ru/threads/gcode/setup/    → content/ru/threads/gcode/01-setup/index.md
/ru/threads/gcode/fan/      → content/ru/threads/gcode/02-fan/index.md
```

**Slug:** Берётся из имени файла БЕЗ номера.
- `01-spiky.md` → slug: `spiky`
- `02-cylinders.md` → slug: `cylinders`
- `03-matrix.md` → slug: `matrix`

**Номер в имени файла** — только для сортировки (не показывается в UI).

**Порядок сортировки:** Обратный — чем **меньше** цифра, тем **ниже** в треде (свежие посты сверху).
- `01-spiky` — самый нижний пост (самый старый)
- `06-cellular` — самый верхний пост (самый свежий)

Цифры временные, их можно менять для изменения порядка. Не нужно переименовывать папки с картинками — они именуются без цифр.

## 📝 Front matter

**Все поля опциональны.**

### Минимальный валидный пост
```yaml
---
# можно вообще пустой front matter или даже без него
---

Какой-то текст или картинки.
```

### Полный пример
```yaml
---
title: "Настройка вентилятора"
date: 2026-01-30
images:
  - /assets/threads/gcode/fan/before.jpg
  - /assets/threads/gcode/fan/after.jpg
---

Заменил вентилятор на Noctua. Стало тише!
```

### Поля

| Поле | Тип | Обязательно | Описание |
|------|-----|-------------|----------|
| `title` | string | нет | Заголовок поста. Если нет — не показывать `<h1>` |
| `date` | YYYY-MM-DD | нет | Дата публикации. Показывать если есть |
| `images` | array | нет | Массив полных путей к картинкам |

**Текст поста** — markdown после front matter. Может отсутствовать (только картинки).

## 🎨 Отображение

### Thread index (`/threads/gcode/`)

**Формат:** Вертикальная лента постов (как Tumblr).

**Каждый пост показывает:**
1. **Заголовок** (если есть `title`) — ссылка на `/threads/gcode/setup/`
2. **Дата** (если есть `date`) — мелким шрифтом, как в текущем блоге
3. **Все картинки** подряд вертикально
4. **Весь текст** (полностью, не excerpt)

**Картинки:**
- Выравнивание: `align: left`
- Масштабирование: без `max-height`, но умное (портретные 9:16 не растягивать во всю ширину)
- Размер: не больше оригинального
- Все картинки подряд вертикально (одна под другой)

**Стили:** Переиспользовать текущие стили блога. Ничего нового не придумывать.

**Разделитель между постами:** Как в текущем блоге (линия или отступ).

---

### Отдельный пост (`/threads/gcode/setup/`)

**Контент:** То же самое, что в ленте (заголовок, дата, картинки, текст).

**Навигация:**

1. **Стрелка назад** — рядом с заголовком (или на том же уровне, если заголовка нет):
   ```
   ← /threads/gcode/
   ```
   Ссылка на thread index.

2. **Навигация между постами** — внизу страницы:
   ```
   ← Предыдущий | Следующий →
   ```
   - Если это первый пост — скрыть "← Предыдущий"
   - Если это последний пост — скрыть "Следующий →"

**Стили:** Как в текущих постах блога.

---

### Ссылки на треды в `/blog/`

**Расположение:** Над списком постов (на самом верху).

**Формат:** Простые ссылки (без карточек):
```
Треды: [G-code] [Shaders] [3D-печать]

─────────────────────────────

[Обычные посты блога...]
```

**Стили:** Как обычные ссылки, можно чуть выделить (жирным или цветом).

**Порядок:** Алфавитный или по дате создания (на усмотрение разработчика).

## 🧭 Техническая реализация

### Eleventy Collections

Создать коллекцию для каждого треда:

```js
// .eleventy.js
eleventyConfig.addCollection("threadGcode", function(collectionApi) {
  return collectionApi.getFilteredByGlob("content/ru/threads/gcode/*.md")
    .sort((a, b) => {
      // ОБРАТНАЯ сортировка по номеру в имени файла
      // Меньше номер → ниже в списке (старые посты внизу)
      const aNum = parseInt(a.filePathStem.match(/(\d+)-/)?.[1] || 0);
      const bNum = parseInt(b.filePathStem.match(/(\d+)-/)?.[1] || 0);
      return bNum - aNum; // Обратный порядок!
    });
});
```

### Slug генерация

Убрать номер из slug:
```js
// Пример permalink в front matter или data file
permalink: "/ru/threads/gcode/{{ page.fileSlug | removeNumber }}/"

// Фильтр removeNumber:
eleventyConfig.addFilter("removeNumber", function(slug) {
  return slug.replace(/^\d+-/, '');
});
```

### Passthrough для картинок

```js
eleventyConfig.addPassthroughCopy({
  "assets/threads/**/*": "assets/threads",
});
```

**Важно:** Картинки хранятся сразу в `assets/threads/`, не в `content/`. Это предотвращает дублирование для разных языков.

### Получение списка тредов

```js
eleventyConfig.addCollection("allThreads", function(collectionApi) {
  const threads = new Set();
  collectionApi.getFilteredByGlob("content/ru/threads/*/*.md")
    .forEach(post => {
      const match = post.filePathStem.match(/threads\/([^\/]+)\//);
      if (match) threads.add(match[1]);
    });
  return Array.from(threads).sort();
});
```

## 📐 Картинки: умное масштабирование

**Проблема:** Портретные фото (9:16) не должны растягиваться во всю ширину блока.

**Решение:**
```css
.thread-image {
  max-width: 100%;
  height: auto;
  display: block;
  margin: 1rem 0;
}

/* Портретные — выравниваем влево, ограничиваем ширину */
.thread-image[data-orientation="portrait"] {
  max-width: 400px; /* или другое разумное значение */
}
```

**Определение ориентации:** Можно через JS или задавать вручную в front matter (если нужно).
