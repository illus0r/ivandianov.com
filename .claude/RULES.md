# Правила работы с сайтом ivandianov.com

## Перед началом работы

**ОБЯЗАТЕЛЬНО:** Прочитай этот файл перед любыми изменениями сайта.

## Рабочий процесс

### 1. Добавление контента

При добавлении нового контента (статьи, посты, проекты):
- **Автоматически создавай переводы** на оба языка (ru/en)
- Сохраняй структуру frontmatter идентичной
- Переводи заголовки, описания, метаданные
- Контент размещай в соответствующих языковых папках: `content/ru/` и `content/en/`

### 2. Проверка ссылок

После сборки сайта (`npm run build`):
- **Автоматически запускай проверку ссылок**
- Команда: `npx linkinator _site --recurse --skip "node_modules|.git"`
- Если находишь broken links: **НЕ ЧИНИ автоматически**
- **ПРЕДУПРЕДИ** пользователя о найденных проблемах
- Предложи варианты исправления

### 3. Типичные команды

```bash
# Разработка с hot reload
npm run serve

# Сборка
npm run build

# Проверка ссылок (требует запущенный сервер)
# Терминал 1: cd _site && python3 -m http.server 8080
# Терминал 2: npx linkinator http://localhost:8080 --recurse --skip "node_modules|.git"
```

## Структура проекта

- `.eleventy.js` — конфигурация Eleventy
- `content/` — контент сайта (ru/en)
- `_includes/` — шаблоны
- `_site/` — собранный сайт (не коммитить)
- `assets/` — статические ресурсы

## Важные принципы

- Всегда тестируй локально перед коммитом
- Проверяй оба языка после изменений
- Сохраняй консистентность между ru/en версиями
- Следи за производительностью сборки
